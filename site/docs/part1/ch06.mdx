# 6장: 계획 수립

## 계획 수립 패턴 개요

지능적인 행동에는 즉각적인 입력에 단순히 반응하는 것 이상이 요구되는 경우가 많습니다. 선견지명을 발휘하고, 복잡한 작업을 더 작고 관리하기 쉬운 단계로 분해하며, 원하는 결과를 달성하기 위한 전략을 수립해야 합니다. 바로 여기서 계획 수립(Planning) 패턴이 등장합니다. 계획 수립의 핵심은 에이전트 또는 에이전트 시스템이 초기 상태에서 목표 상태로 이동하기 위한 일련의 행동을 수립하는 능력입니다.

AI 맥락에서 계획 수립 에이전트는 복잡한 목표를 위임받는 전문가로 생각하면 유용합니다. "팀 워크숍을 조직해 주세요"라고 요청할 때, 여러분은 무엇(what)—목표와 제약 조건—을 정의하는 것이지, 어떻게(how)를 정의하는 것이 아닙니다. 에이전트의 핵심 과제는 그 목표를 향한 경로를 자율적으로 설계하는 것입니다. 에이전트는 먼저 초기 상태(예: 예산, 참가자 수, 원하는 날짜)와 목표 상태(성공적으로 예약된 워크숍)를 파악한 다음, 이 두 상태를 연결하는 최적의 행동 순서를 찾아내야 합니다. 계획은 미리 정해진 것이 아니라 요청에 대한 응답으로 만들어집니다.

이 과정의 특징은 적응성입니다. 초기 계획은 단지 출발점일 뿐, 고정된 대본이 아닙니다. 에이전트의 진정한 능력은 새로운 정보를 통합하고 장애물을 우회하는 데 있습니다. 예를 들어, 선호하는 장소를 사용할 수 없거나 선택한 케이터링 업체가 모두 예약된 경우, 유능한 에이전트는 단순히 실패하지 않습니다. 에이전트는 적응합니다. 새로운 제약 조건을 파악하고, 옵션을 재평가하며, 대안적인 장소나 날짜를 제안하는 등 새로운 계획을 수립합니다.

그러나 유연성과 예측 가능성 사이의 트레이드오프를 인식하는 것이 중요합니다. 동적 계획 수립은 특정한 도구이지, 보편적인 해결책이 아닙니다. 문제의 해결책이 이미 잘 알려져 있고 반복 가능한 경우, 에이전트를 미리 결정된 고정 워크플로로 제한하는 것이 더 효과적입니다. 이 접근 방식은 에이전트의 자율성을 제한하여 불확실성과 예측 불가능한 행동의 위험을 줄이고, 신뢰할 수 있고 일관된 결과를 보장합니다. 따라서 계획 수립 에이전트와 단순 작업 실행 에이전트 중 무엇을 선택할지는 단 하나의 질문에 달려 있습니다: '어떻게'를 발견해야 하는가, 아니면 이미 알려져 있는가?

## 실용적 응용 및 사용 사례

계획 수립(Planning) 패턴은 자율 시스템의 핵심 계산 프로세스로, 에이전트가 특히 동적이거나 복잡한 환경에서 지정된 목표를 달성하기 위한 일련의 행동을 합성할 수 있게 합니다. 이 프로세스는 고수준 목표를 이산적이고 실행 가능한 단계로 구성된 구조화된 계획으로 변환합니다.

절차적 작업 자동화 영역에서 계획 수립은 복잡한 워크플로를 오케스트레이션하는 데 사용됩니다. 예를 들어, 신규 직원 온보딩과 같은 비즈니스 프로세스는 시스템 계정 생성, 교육 모듈 배정, 다양한 부서와의 조율과 같은 하위 작업의 지향적 순서로 분해될 수 있습니다. 에이전트는 이러한 단계를 논리적 순서로 실행하고, 필요한 도구를 호출하거나 다양한 시스템과 상호작용하여 의존성을 관리하는 계획을 생성합니다.

로보틱스와 자율 항법 영역에서 계획 수립은 상태 공간 탐색에 필수적입니다. 물리적 로봇이든 가상 엔티티든, 시스템은 초기 상태에서 목표 상태로 전환하기 위한 경로나 행동 순서를 생성해야 합니다. 이는 장애물 회피나 교통 규정 준수와 같은 환경 제약 조건을 준수하면서 시간이나 에너지 소비와 같은 지표를 최적화하는 것을 포함합니다.

이 패턴은 구조화된 정보 합성에도 중요합니다. 연구 보고서와 같은 복잡한 출력을 생성하는 작업을 받으면, 에이전트는 정보 수집, 데이터 요약, 콘텐츠 구조화, 반복적 정제를 위한 별개의 단계를 포함하는 계획을 수립할 수 있습니다. 마찬가지로, 다단계 문제 해결을 포함하는 고객 지원 시나리오에서 에이전트는 진단, 솔루션 구현, 에스컬레이션을 위한 체계적인 계획을 수립하고 따를 수 있습니다.

본질적으로 계획 수립 패턴은 에이전트가 단순한 반응적 행동을 넘어 목표 지향적 행동으로 나아갈 수 있게 합니다. 상호 의존적인 연산의 일관된 순서를 필요로 하는 문제를 해결하는 데 필요한 논리적 프레임워크를 제공합니다.

## 실습 코드 (Crew AI)

다음 섹션은 Crew AI 프레임워크를 사용한 계획 수립(Planner) 패턴 구현을 시연합니다. 이 패턴은 복잡한 쿼리를 처리하기 위해 먼저 다단계 계획을 수립하고 그 계획을 순차적으로 실행하는 에이전트를 포함합니다.

```python
import os
from dotenv import load_dotenv
from crewai import Agent, Task, Crew, Process
from langchain_openai import ChatOpenAI

# Load environment variables from .env file for security
load_dotenv()
```

```python
# 1. Explicitly define the language model for clarity
llm = ChatOpenAI(model="gpt-4-turbo")

# 2. Define a clear and focused agent
planner_writer_agent = Agent(
   role='Article Planner and Writer',
   goal='Plan and then write a concise, engaging summary on a
specified topic.',
   backstory=(
       'You are an expert technical writer and content strategist. '
       'Your strength lies in creating a clear, actionable plan
before writing, '
       'ensuring the final summary is both informative and easy to
digest.'
   ),
   verbose=True,
   allow_delegation=False,
   llm=llm # Assign the specific LLM to the agent
)

# 3. Define a task with a more structured and specific expected
output
topic = "The importance of Reinforcement Learning in AI"
high_level_task = Task(
   description=(
       f"1. Create a bullet-point plan for a summary on the topic:
'{topic}'.\n"
       f"2. Write the summary based on your plan, keeping it around
200 words."
   ),
   expected_output=(
       "A final report containing two distinct sections:\n\n"
       "### Plan\n"
       "- A bulleted list outlining the main points of the
summary.\n\n"
       "### Summary\n"
       "- A concise and well-structured summary of the topic."
   ),
   agent=planner_writer_agent,
)

# Create the crew with a clear process
crew = Crew(
   agents=[planner_writer_agent],
   tasks=[high_level_task],
   process=Process.sequential,
```

```python
)

# Execute the task
print("## Running the planning and writing task ##")
result = crew.kickoff()

print("\n\n---\n## Task Result ##\n---")
print(result)
```

이 코드는 CrewAI 라이브러리를 사용하여 주어진 주제에 대한 요약을 계획하고 작성하는 AI 에이전트를 생성합니다. Crew.ai와 langchain_openai를 포함한 필요한 라이브러리를 임포트하고 .env 파일에서 환경 변수를 로드하는 것으로 시작합니다. ChatOpenAI 언어 모델이 에이전트와 함께 사용하도록 명시적으로 정의됩니다. `planner_writer_agent`라는 에이전트가 특정 역할과 목표—계획을 수립한 다음 간결한 요약을 작성하는 것—로 생성됩니다. 에이전트의 배경 이야기는 계획 수립과 기술 작문에 대한 전문성을 강조합니다. 작업은 먼저 계획을 세운 다음 "AI에서 강화학습의 중요성"이라는 주제에 대한 요약을 작성하는 명확한 설명과 예상 출력 형식으로 정의됩니다. 에이전트와 작업으로 크루(Crew)가 구성되고 순차적으로 처리되도록 설정됩니다. 마지막으로 `crew.kickoff()` 메서드가 호출되어 정의된 작업을 실행하고 결과가 출력됩니다.

## Google DeepResearch

Google Gemini DeepResearch(그림 1 참조)는 자율적인 정보 검색 및 합성을 위해 설계된 에이전트 기반 시스템입니다. 복잡한 주제를 체계적으로 탐색하기 위해 Google Search를 동적으로 반복 쿼리하는 다단계 에이전틱 파이프라인을 통해 작동합니다. 이 시스템은 방대한 웹 기반 소스를 처리하고, 수집된 데이터의 관련성과 지식 격차를 평가하며, 이를 해결하기 위한 후속 검색을 수행하도록 설계되었습니다. 최종 출력은 검증된 정보를 원본 출처 인용이 포함된 구조화된 다중 페이지 요약으로 통합합니다.

더 나아가, 이 시스템의 작동 방식은 단일 쿼리-응답 이벤트가 아니라 관리되는 장기 실행 프로세스입니다. 사용자의 프롬프트를 다중 포인트 연구 계획으로 분해하는 것으로 시작하며(그림 1 참조), 이 계획은 실행 전에 사용자가 검토하고 수정할 수 있도록 제시됩니다. 이를 통해 실행 전에 연구 방향을 협력적으로 설정할 수 있습니다. 계획이 승인되면 에이전틱 파이프라인이 반복적인 검색-분석 루프를 시작합니다. 이는 미리 정해진 일련의 검색을 실행하는 것에 그치지 않습니다. 에이전트는 수집하는 정보를 기반으로 쿼리를 동적으로 수립하고 정제하여, 지식 격차를 능동적으로 파악하고, 데이터 포인트를 검증하며, 불일치를 해결합니다.

이 접근 방식의 효율성은 수동 연구에서 핵심적인 병목인 반복적인 검색-필터링 사이클의 자동화에서 비롯됩니다. 시스템이 비교 가능한 시간 내에 인간 연구자가 일반적으로 처리할 수 있는 것보다 더 많은 양과 다양한 정보 소스를 처리할 수 있는 역량을 통해 포괄성이 달성됩니다. 이 더 넓은 분석 범위는 선택 편향의 가능성을 줄이고, 덜 명확하지만 잠재적으로 중요한 정보를 발견할 가능성을 높여, 주제에 대한 더 견고하고 잘 뒷받침된 이해로 이어집니다.

## OpenAI Deep Research API

OpenAI Deep Research API는 복잡한 연구 작업을 자동화하도록 설계된 특화된 도구입니다. 실제 소스에서 독립적으로 추론하고, 계획을 수립하며, 정보를 합성할 수 있는 고급 에이전틱 모델을 활용합니다. 단순한 질의응답 모델과 달리, 고수준 쿼리를 받아 자율적으로 하위 질문으로 분해하고, 내장 도구를 사용하여 웹 검색을 수행하며, 구조화된 인용 풍부 최종 보고서를 제공합니다. API는 이 전체 프로세스에 대한 직접적인 프로그래매틱 접근을 제공하며, 집필 시점에는 고품질 합성을 위한 o3-deep-research-2025-06-26 모델과 지연 시간에 민감한 애플리케이션을 위한 더 빠른 o4-mini-deep-research-2025-06-26 모델과 같은 모델을 사용합니다.

Deep Research API는 그렇지 않으면 수 시간의 수동 연구가 필요한 작업을 자동화하여, 비즈니스 전략, 투자 결정 또는 정책 권고를 지원하기에 적합한 전문적 수준의 데이터 기반 보고서를 제공하기 때문에 유용합니다. 주요 이점은 다음과 같습니다:

- **구조화된 인용 출력(Structured, Cited Output)**: 소스 메타데이터에 연결된 인라인 인용이 포함된 잘 정리된 보고서를 생성하여 주장이 검증 가능하고 데이터에 기반하도록 합니다.
- **투명성(Transparency)**: ChatGPT의 추상화된 프로세스와 달리, API는 에이전트의 추론, 실행된 특정 웹 검색 쿼리, 실행된 코드를 포함한 모든 중간 단계를 공개합니다. 이를 통해 상세한 디버깅, 분석, 최종 답변이 어떻게 구성되었는지에 대한 깊은 이해가 가능합니다.
- **확장성(Extensibility)**: 모델 컨텍스트 프로토콜(MCP)을 지원하여 개발자가 에이전트를 개인 지식 베이스 및 내부 데이터 소스에 연결하고, 공개 웹 연구와 독점 정보를 혼합할 수 있습니다.

API를 사용하려면 `client.responses.create` 엔드포인트에 요청을 보내고, 모델, 입력 프롬프트, 에이전트가 사용할 수 있는 도구를 지정합니다. 입력에는 일반적으로 에이전트의 페르소나와 원하는 출력 형식을 정의하는 `system_message`와 `user_query`가 포함됩니다. `web_search_preview` 도구를 반드시 포함해야 하며, 내부 데이터를 위해 `code_interpreter`나 사용자 정의 MCP 도구(10장 참조)를 선택적으로 추가할 수 있습니다.

```python
from openai import OpenAI

# Initialize the client with your API key
client = OpenAI(api_key="YOUR_OPENAI_API_KEY")

# Define the agent's role and the user's research question
system_message = """You are a professional researcher preparing a
structured, data-driven report.
Focus on data-rich insights, use reliable sources, and include inline
citations."""
user_query = "Research the economic impact of semaglutide on global
healthcare systems."

# Create the Deep Research API call
response = client.responses.create(
 model="o3-deep-research-2025-06-26",
 input=[
   {
     "role": "developer",
     "content": [{"type": "input_text", "text": system_message}]
   },
   {
     "role": "user",
     "content": [{"type": "input_text", "text": user_query}]
   }
 ],
 reasoning={"summary": "auto"},
 tools=[{"type": "web_search_preview"}]
)
```

```python
# Access and print the final report from the response
final_report = response.output[-1].content[0].text
print(final_report)

# --- ACCESS INLINE CITATIONS AND METADATA ---
print("--- CITATIONS ---")
annotations = response.output[-1].content[0].annotations

if not annotations:
   print("No annotations found in the report.")
else:
   for i, citation in enumerate(annotations):
       # The text span the citation refers to
       cited_text =
final_report[citation.start_index:citation.end_index]

       print(f"Citation {i+1}:")
       print(f"  Cited Text: {cited_text}")
       print(f"  Title: {citation.title}")
       print(f"  URL: {citation.url}")
       print(f"  Location: chars
{citation.start_index}–{citation.end_index}")
print("\n" + "="*50 + "\n")

# --- INSPECT INTERMEDIATE STEPS ---
print("--- INTERMEDIATE STEPS ---")

# 1. Reasoning Steps: Internal plans and summaries generated by the
model.
try:
   reasoning_step = next(item for item in response.output if
item.type == "reasoning")
   print("\n[Found a Reasoning Step]")
   for summary_part in reasoning_step.summary:
       print(f"  - {summary_part.text}")
except StopIteration:
   print("\nNo reasoning steps found.")

# 2. Web Search Calls: The exact search queries the agent executed.
try:
   search_step = next(item for item in response.output if item.type
== "web_search_call")
   print("\n[Found a Web Search Call]")
   print(f"  Query Executed: '{search_step.action['query']}'")
   print(f"  Status: {search_step.status}")
except StopIteration:
   print("\nNo web search steps found.")

# 3. Code Execution: Any code run by the agent using the code
interpreter.
try:
   code_step = next(item for item in response.output if item.type ==
"code_interpreter_call")
   print("\n[Found a Code Execution Step]")
   print("  Code Input:")
   print(f"  ```python\n{code_step.input}\n  ```")

```python
   print("  Code Output:")
   print(f"  {code_step.output}")
except StopIteration:
   print("\nNo code execution steps found.")
```

이 코드 스니펫은 OpenAI API를 사용하여 "Deep Research" 작업을 수행합니다. OpenAI 클라이언트를 API 키로 초기화하는 것으로 시작합니다(인증에 필수적입니다). 그런 다음 AI 에이전트의 역할을 전문 연구원으로 정의하고 세마글루타이드의 경제적 영향에 관한 사용자의 연구 질문을 설정합니다. 코드는 정의된 시스템 메시지와 사용자 쿼리를 입력으로 제공하면서 o3-deep-research-2025-06-26 모델에 대한 API 호출을 구성합니다. 또한 추론의 자동 요약을 요청하고 웹 검색 기능을 활성화합니다. API 호출 후, 최종 생성된 보고서를 추출하고 출력합니다.

이후 보고서의 주석에서 인용된 텍스트, 제목, URL, 보고서 내 위치를 포함한 인라인 인용 및 메타데이터에 접근하고 표시하려고 시도합니다. 마지막으로, 추론 단계, 웹 검색 호출(실행된 쿼리 포함), 코드 인터프리터가 사용된 경우 코드 실행 단계와 같이 모델이 취한 중간 단계에 대한 세부 정보를 검사하고 출력합니다.

## 한눈에 보기

**무엇이 문제인가**: 복잡한 문제는 단일 행동으로 해결할 수 없는 경우가 많으며 원하는 결과를 달성하기 위한 선견지명이 필요합니다. 구조화된 접근 방식 없이는 에이전틱 시스템이 여러 단계와 의존성을 포함하는 복잡한 요청을 처리하는 데 어려움을 겪습니다. 이로 인해 고수준 목표를 관리 가능한 일련의 더 작고 실행 가능한 작업으로 분해하기 어렵습니다. 결과적으로 시스템은 효과적으로 전략화하지 못하여, 복잡한 목표에 직면했을 때 불완전하거나 잘못된 결과로 이어집니다.

**왜 계획 수립 패턴인가**: 계획 수립(Planning) 패턴은 에이전틱 시스템이 먼저 목표를 해결하기 위한 일관된 계획을 수립하는 표준화된 솔루션을 제공합니다. 고수준 목표를 더 작고 실행 가능한 단계 또는 하위 목표의 순서로 분해하는 것을 포함합니다. 이를 통해 시스템은 복잡한 워크플로를 관리하고, 다양한 도구를 오케스트레이션하며, 논리적 순서로 의존성을 처리할 수 있습니다. LLM은 방대한 학습 데이터를 기반으로 그럴듯하고 효과적인 계획을 생성할 수 있어 이에 특히 적합합니다. 이 구조화된 접근 방식은 단순한 반응적 에이전트를 복잡한 목표를 향해 사전적으로 작업하고 필요한 경우 계획을 적응시킬 수 있는 전략적 실행자로 변환합니다.

**경험 법칙**: 사용자의 요청이 단일 행동이나 도구로 처리하기에 너무 복잡한 경우 이 패턴을 사용하십시오. 상세한 연구 보고서 생성, 신규 직원 온보딩, 경쟁 분석 실행과 같은 다단계 프로세스를 자동화하는 데 이상적입니다. 최종적으로 합성된 결과에 도달하기 위해 상호 의존적인 연산의 순서가 필요한 작업이 있을 때마다 계획 수립(Planning) 패턴을 적용하십시오.

## 핵심 정리

- **계획 수립(Planning)** 은 에이전트가 복잡한 목표를 실행 가능하고 순차적인 단계로 분해할 수 있게 합니다.
- 다단계 작업 처리, 워크플로 자동화, 복잡한 환경 탐색에 필수적입니다.
- LLM은 작업 설명을 기반으로 단계별 접근 방식을 생성하여 계획 수립을 수행할 수 있습니다.
- 계획 단계를 필요로 하도록 명시적으로 프롬프트를 작성하거나 작업을 설계하면 에이전트 프레임워크에서 이러한 행동을 유도합니다.
- Google Deep Research는 에이전트가 Google Search를 도구로 사용하여 확보한 소스를 우리를 대신하여 분석합니다. 에이전트는 반성하고, 계획을 수립하며, 실행합니다.

## 결론

결론적으로, 계획 수립(Planning) 패턴은 에이전틱 시스템을 단순한 반응적 응답자에서 전략적이고 목표 지향적인 실행자로 끌어올리는 핵심 구성 요소입니다. 현대의 대규모 언어 모델은 고수준 목표를 일관되고 실행 가능한 단계로 자율적으로 분해하는 핵심 능력을 제공합니다. 이 패턴은 CrewAI 에이전트가 작성 계획을 수립하고 따르는 것으로 시연된 간단하고 순차적인 작업 실행부터 더 복잡하고 동적인 시스템까지 확장됩니다. Google DeepResearch 에이전트는 지속적인 정보 수집을 기반으로 적응하고 발전하는 반복적인 연구 계획을 수립하면서 이 고급 응용 사례를 잘 보여줍니다. 궁극적으로, 계획 수립은 복잡한 문제에 대한 인간의 의도와 자동화된 실행 사이의 필수적인 다리를 제공합니다. 문제 해결 접근 방식을 구조화함으로써, 이 패턴은 에이전트가 복잡한 워크플로를 관리하고 포괄적이며 합성된 결과를 제공할 수 있게 합니다.
