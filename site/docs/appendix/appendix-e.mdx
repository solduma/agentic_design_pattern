# 부록 E: CLI의 AI 에이전트

## 소개

오랫동안 정밀하고 명령형(imperative) 명령의 근거지였던 개발자의 커맨드 라인이 심층적인 변혁을 겪고 있습니다. 단순한 셸에서 새로운 종류의 도구인 AI 에이전트 CLI(Command-Line Interface)가 구동하는 지능적이고 협업적인 워크스페이스로 진화하고 있습니다. 이러한 에이전트는 단순히 명령을 실행하는 것을 넘어서, 자연어를 이해하고 전체 코드베이스에 대한 컨텍스트를 유지하며 개발 생명주기의 상당 부분을 자동화하는 복잡한 다단계 작업을 수행할 수 있습니다.

이 가이드는 급성장하는 분야에서 선도적인 네 가지 도구를 심층적으로 살펴보고, 각 도구의 고유한 강점, 이상적인 사용 사례, 그리고 차별화된 철학을 분석하여 어떤 도구가 여러분의 워크플로에 가장 적합한지 판단하는 데 도움을 드립니다. 특정 도구에 대해 제시된 많은 예시 사용 사례들이 다른 에이전트로도 수행될 수 있다는 점에 유의하십시오. 이 도구들 간의 핵심 차별화 요소는 종종 특정 작업에서 달성할 수 있는 결과의 품질, 효율성, 그리고 뉘앙스에 있습니다. 이러한 역량을 측정하기 위해 설계된 특정 벤치마크가 있으며, 이에 대해서는 다음 절에서 논의합니다.

## Claude CLI (Claude Code)

Anthropic의 Claude CLI는 프로젝트 아키텍처에 대한 깊고 전체적인 이해를 갖춘 고수준 코딩 에이전트로 설계되었습니다. 핵심 강점은 "에이전틱(agentic)" 특성으로, 복잡한 다단계 작업을 위해 저장소의 멘탈 모델을 구성할 수 있습니다. 상호작용은 매우 대화적이며, 실행 전에 계획을 설명하는 페어 프로그래밍 세션과 흡사합니다. 이 점에서 대규모 리팩터링이나 광범위한 아키텍처 영향을 수반하는 기능 구현을 다루는 대규모 프로젝트에 종사하는 전문 개발자에게 이상적입니다.

**예시 사용 사례:**

1. **대규모 리팩터링:** "현재 사용자 인증이 세션 쿠키에 의존합니다. 전체 코드베이스를 무상태 JWT를 사용하도록 리팩터링하여 로그인/로그아웃 엔드포인트, 미들웨어, 프론트엔드 토큰 처리를 업데이트하십시오."와 같이 지시할 수 있습니다. Claude는 모든 관련 파일을 읽고 조율된 변경을 수행합니다.

2. **API 통합:** 새로운 날씨 서비스의 OpenAPI 사양을 제공한 후, "이 새 날씨 API를 통합하십시오. API 호출을 처리하는 서비스 모듈을 생성하고, 날씨를 표시하는 새 컴포넌트를 추가하며, 메인 대시보드에 이를 포함하도록 업데이트하십시오."라고 요청할 수 있습니다.

3. **문서 생성:** 문서화가 미흡한 복잡한 모듈을 지정하고 "이 `./src/utils/data_processing.js` 파일을 분석하십시오. 모든 함수에 대해 목적, 매개변수, 반환값을 설명하는 포괄적인 TSDoc 주석을 생성하십시오."라고 요청할 수 있습니다.

Claude CLI는 파일 수집, 코드 구조 분석, 편집 생성 등 핵심 개발 작업을 위한 내장 도구를 갖춘 특화된 코딩 어시스턴트로 기능합니다. Git과의 깊은 통합으로 직접적인 브랜치 및 커밋 관리가 가능합니다. 에이전트의 확장성은 MCP(Multi-tool Control Protocol)를 통해 중재되어, 사용자가 커스텀 도구를 정의하고 통합할 수 있습니다. 이를 통해 프라이빗 API와의 상호작용, 데이터베이스 쿼리, 프로젝트별 스크립트 실행이 가능합니다. 이러한 아키텍처는 개발자를 에이전트의 기능 범위를 결정하는 중재자로 위치시키며, Claude를 사용자 정의 툴링으로 보강된 추론 엔진으로 효과적으로 규정합니다.

## Gemini CLI

Google의 Gemini CLI는 강력함과 접근성을 위해 설계된 다재다능한 오픈소스 AI 에이전트입니다. 고급 Gemini 2.5 Pro 모델, 방대한 컨텍스트 윈도우, 멀티모달 기능(이미지 및 텍스트 처리)으로 두각을 나타냅니다. 오픈소스 특성, 관대한 무료 티어, "추론 및 행동(Reason and Act)" 루프는 이를 투명하고 제어 가능한 범용 도구로 만들어, 취미 개발자부터 엔터프라이즈 개발자까지, 특히 Google Cloud 에코시스템 내의 광범위한 사용자층에 탁월합니다.

**예시 사용 사례:**

1. **멀티모달 개발:** 디자인 파일의 웹 컴포넌트 스크린샷을 제공하고(`gemini describe component.png`) "이 모습과 정확히 같은 React 컴포넌트를 구축하는 HTML과 CSS 코드를 작성하십시오. 반응형으로 만드십시오."라고 지시할 수 있습니다.

2. **클라우드 리소스 관리:** 내장된 Google Cloud 통합을 사용하여 "프로덕션 프로젝트에서 1.28 이전 버전을 실행 중인 모든 GKE 클러스터를 찾고, 하나씩 업그레이드하는 gcloud 명령을 생성하십시오."라고 명령할 수 있습니다.

3. **엔터프라이즈 도구 통합 (MCP를 통해):** 개발자가 회사의 내부 HR API에 연결하는 `get-employee-details`라는 커스텀 도구를 Gemini에 제공합니다. 프롬프트는 "신입 직원을 위한 환영 문서를 작성하십시오. 먼저 `get-employee-details --id=E90210` 도구를 사용하여 이름과 팀을 가져온 다음, 해당 정보로 `welcome_template.md`를 채우십시오."입니다.

4. **대규모 리팩터링:** 개발자가 대규모 Java 코드베이스를 더 이상 사용되지 않는 로깅 라이브러리에서 새로운 구조화된 로깅 프레임워크로 교체해야 합니다. `'src/main/java'` 디렉터리의 모든 `*.java` 파일을 읽고, 각 파일에서 `'org.apache.log4j'` 임포트와 `Logger` 클래스를 `'org.slf4j.Logger'`와 `LoggerFactory`로 교체하며, 로거 인스턴스화와 모든 `.info()`, `.debug()`, `.error()` 호출을 키-값 쌍을 사용하는 새 구조화된 형식으로 재작성하는 프롬프트로 Gemini를 사용할 수 있습니다.

Gemini CLI는 환경과 상호작용할 수 있는 내장 도구 모음을 갖추고 있습니다. 파일 시스템 작업(읽기 및 쓰기), 명령 실행을 위한 셸 도구, 웹 페치 및 검색을 통한 인터넷 접근 도구가 포함됩니다. 더 넓은 컨텍스트를 위해 여러 파일을 한 번에 읽는 특화 도구와 이후 세션을 위해 정보를 저장하는 메모리 도구를 사용합니다. 이 기능은 안전한 기반 위에 구축됩니다. 샌드박싱은 위험을 방지하기 위해 모델의 행동을 격리하고, MCP 서버는 Gemini가 로컬 환경이나 다른 API에 안전하게 연결할 수 있는 브리지 역할을 합니다.

## Aider

Aider는 파일에 직접 작업하고 변경 사항을 Git에 커밋함으로써 진정한 페어 프로그래머로 기능하는 오픈소스 AI 코딩 어시스턴트입니다. 정의적 특징은 직접성입니다. 편집을 적용하고, 테스트를 실행하여 유효성을 검사하며, 모든 성공적인 변경을 자동으로 커밋합니다. 모델 불가지론적(model-agnostic) 설계로 비용과 기능에 대한 완전한 제어권을 사용자에게 부여합니다. Git 중심 워크플로는 효율성, 제어, 그리고 모든 코드 수정에 대한 투명하고 감사 가능한 추적을 중시하는 개발자에게 완벽합니다.

**예시 사용 사례:**

1. **테스트 주도 개발(TDD):** 개발자는 "숫자의 팩토리얼을 계산하는 함수에 대한 실패하는 테스트를 생성하십시오."라고 말할 수 있습니다. Aider가 테스트를 작성하고 실패한 후, 다음 프롬프트는 "이제 테스트를 통과하는 코드를 작성하십시오."입니다. Aider가 함수를 구현하고 테스트를 다시 실행하여 확인합니다.

2. **정밀한 버그 수정:** 버그 보고서가 주어지면, "billing.py의 `calculate_total` 함수가 윤년에 실패합니다. 파일을 컨텍스트에 추가하고, 버그를 수정하며, 기존 테스트 스위트에 대해 수정 사항을 검증하십시오."라고 Aider에게 지시할 수 있습니다.

3. **의존성 업데이트:** "우리 프로젝트가 오래된 버전의 `requests` 라이브러리를 사용합니다. 모든 Python 파일을 검토하여 임포트 문과 더 이상 사용되지 않는 함수 호출을 최신 버전과 호환되도록 업데이트하고, `requirements.txt`를 업데이트하십시오."라고 지시할 수 있습니다.

## GitHub Copilot CLI

GitHub Copilot CLI는 인기 있는 AI 페어 프로그래머를 터미널로 확장하며, 주요 강점은 GitHub 에코시스템과의 네이티브하고 깊은 통합입니다. GitHub 내의 프로젝트 컨텍스트를 이해합니다. 에이전트 기능을 통해 GitHub 이슈를 할당받고, 수정 작업을 수행하며, 인간 검토를 위한 풀 리퀘스트를 제출할 수 있습니다.

**예시 사용 사례:**

1. **자동화된 이슈 해결:** 매니저가 버그 티켓(예: "이슈 #123: 페이지네이션의 오프-바이-원 오류 수정")을 Copilot 에이전트에 할당합니다. 에이전트는 새 브랜치를 체크아웃하고, 코드를 작성하며, 이슈를 참조하는 풀 리퀘스트를 제출합니다. 이 모든 것이 수동 개발자 개입 없이 이루어집니다.

2. **저장소 인식 Q&A:** 팀의 신규 개발자는 "이 저장소에서 데이터베이스 연결 로직이 어디에 정의되어 있으며, 어떤 환경 변수가 필요합니까?"라고 질문할 수 있습니다. Copilot CLI는 전체 저장소에 대한 인식을 활용하여 파일 경로와 함께 정확한 답변을 제공합니다.

3. **셸 명령 도우미:** 복잡한 셸 명령이 확실하지 않을 때, 사용자는 `gh? find all files larger than 50MB, compress them, and place them in an archive folder`라고 요청할 수 있습니다. Copilot은 작업을 수행하는 데 필요한 정확한 셸 명령을 생성합니다.

## Terminal-Bench: CLI의 AI 에이전트를 위한 벤치마크

Terminal-Bench는 커맨드 라인 인터페이스 내에서 복잡한 작업을 실행하는 AI 에이전트의 숙련도를 평가하기 위해 설계된 새로운 평가 프레임워크입니다. 터미널은 텍스트 기반의 샌드박스 특성으로 인해 AI 에이전트 운영의 최적 환경으로 식별됩니다. 초기 릴리스인 Terminal-Bench-Core-v0은 과학 워크플로와 데이터 분석 같은 영역에 걸친 80개의 수동 큐레이션 작업으로 구성됩니다. 공정한 비교를 보장하기 위해, 다양한 언어 모델의 표준화된 테스트베드로 기능하는 미니멀리스트 에이전트인 Terminus가 개발되었습니다. 이 프레임워크는 확장성을 위해 설계되어, 컨테이너화 또는 직접 연결을 통해 다양한 에이전트의 통합을 허용합니다. 향후 개발 계획에는 대규모 병렬 평가 활성화와 확립된 벤치마크 통합이 포함됩니다. 이 프로젝트는 작업 확장과 협업적 프레임워크 향상을 위한 오픈소스 기여를 장려합니다.

## 결론

이러한 강력한 AI 커맨드 라인 에이전트의 등장은 소프트웨어 개발의 근본적인 전환을 의미하며, 터미널을 역동적이고 협업적인 환경으로 변모시킵니다. 살펴본 바와 같이, 단 하나의 "최고" 도구는 없습니다. 대신 각 에이전트가 특화된 강점을 제공하는 활기찬 에코시스템이 형성되고 있습니다. 이상적인 선택은 전적으로 개발자의 요구에 달려 있습니다. 복잡한 아키텍처 작업에는 Claude, 다재다능한 멀티모달 문제 해결에는 Gemini, Git 중심의 직접적인 코드 편집에는 Aider, GitHub 워크플로에의 원활한 통합에는 GitHub Copilot이 적합합니다. 이러한 도구들이 계속 발전함에 따라, 이를 활용하는 숙련도는 개발자가 소프트웨어를 빌드하고 디버그하며 관리하는 방식을 근본적으로 변화시키는 필수 기술이 될 것입니다.
