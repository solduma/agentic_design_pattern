# 9장: 학습과 적응

학습과 적응(Learning and Adaptation)은 인공지능 에이전트의 역량을 향상시키는 데 있어 핵심적인 역할을 합니다. 이러한 과정을 통해 에이전트는 사전에 정의된 매개변수를 넘어 진화할 수 있으며, 경험과 환경과의 상호작용을 통해 자율적으로 성능을 향상시킬 수 있습니다. 학습하고 적응함으로써 에이전트는 지속적인 수동 개입 없이도 새로운 상황을 효과적으로 처리하고 성능을 최적화할 수 있습니다. 이 장에서는 에이전트 학습 및 적응의 원리와 메커니즘을 상세히 살펴봅니다.

## 전체 개요

에이전트는 새로운 경험과 데이터를 바탕으로 사고, 행동, 지식을 변화시켜 학습하고 적응합니다. 이를 통해 에이전트는 단순히 지시를 따르는 것에서 벗어나 시간이 지남에 따라 더 스마트해질 수 있습니다.

- **강화학습(Reinforcement Learning)**: 에이전트가 행동을 시도하고 긍정적 결과에 대한 보상과 부정적 결과에 대한 페널티를 받아 변화하는 상황에서 최적의 행동을 학습합니다. 로봇을 제어하거나 게임을 플레이하는 에이전트에 유용합니다.
- **지도 학습(Supervised Learning)**: 에이전트가 레이블이 붙은 예시로부터 학습하여 입력을 원하는 출력에 연결함으로써 의사결정 및 패턴 인식 같은 작업을 가능하게 합니다. 이메일을 분류하거나 트렌드를 예측하는 에이전트에 이상적입니다.
- **비지도 학습(Unsupervised Learning)**: 에이전트가 레이블 없는 데이터에서 숨겨진 연결과 패턴을 발견하여 인사이트, 조직화, 환경의 정신적 지도 생성에 도움을 줍니다. 특정 지침 없이 데이터를 탐색하는 에이전트에 유용합니다.
- **LLM 기반 에이전트를 활용한 퓨샷 학습(Few-Shot Learning)/제로샷 학습(Zero-Shot Learning)**: LLM을 활용하는 에이전트는 최소한의 예시나 명확한 지시만으로 새로운 작업에 빠르게 적응하여 새로운 명령이나 상황에 신속하게 대응할 수 있습니다.
- **온라인 학습(Online Learning)**: 에이전트가 새로운 데이터로 지식을 지속적으로 업데이트하며, 동적 환경에서의 실시간 반응과 지속적인 적응에 필수적입니다. 연속적인 데이터 스트림을 처리하는 에이전트에 매우 중요합니다.
- **기억 기반 학습(Memory-Based Learning)**: 에이전트가 과거 경험을 상기하여 유사한 상황에서 현재의 행동을 조정함으로써 맥락 인식과 의사결정 능력을 향상시킵니다. 기억 회상 기능을 갖춘 에이전트에 효과적입니다.

에이전트는 학습을 통해 전략, 이해, 목표를 변화시켜 적응합니다. 이는 예측 불가능하거나 변화하는 환경 또는 새로운 환경에 처한 에이전트에게 매우 중요합니다.

## 근위 정책 최적화(Proximal Policy Optimization, PPO)

근위 정책 최적화(PPO)는 로봇의 관절 제어나 게임 내 캐릭터 조작과 같이 연속적인 행동 범위를 갖는 환경에서 에이전트를 훈련하는 데 사용되는 강화학습 알고리즘입니다. 주요 목표는 에이전트의 의사결정 전략, 즉 정책(policy)을 신뢰성 있고 안정적으로 개선하는 것입니다.

PPO의 핵심 아이디어는 에이전트의 정책을 작고 신중하게 업데이트하는 것입니다. 성능 붕괴를 야기할 수 있는 급격한 변화를 피합니다. 작동 방식은 다음과 같습니다.

1. **데이터 수집**: 에이전트가 현재 정책을 사용하여 환경과 상호작용하고(예: 게임 플레이) 경험 배치(상태, 행동, 보상)를 수집합니다.
2. **"대리" 목표 평가**: PPO는 잠재적인 정책 업데이트가 예상 보상을 어떻게 변화시킬지 계산합니다. 그러나 이 보상을 단순히 극대화하는 대신 특수한 "클리핑된(clipped)" 목적 함수를 사용합니다.
3. **"클리핑" 메커니즘**: 이것이 PPO 안정성의 핵심입니다. 현재 정책 주변에 "신뢰 영역(trust region)" 또는 안전 구역을 만들어 알고리즘이 현재 전략과 너무 다른 업데이트를 수행하지 못하도록 합니다. 이 클리핑은 안전 브레이크처럼 작동하여 에이전트가 학습을 무효화할 수 있는 크고 위험한 도약을 하지 않도록 합니다.

요약하면, PPO는 알려진 작동하는 전략에 가깝게 유지하면서 성능을 개선하는 균형을 맞추어, 훈련 중 치명적인 실패를 방지하고 더 안정적인 학습으로 이어집니다.

## 직접 선호 최적화(Direct Preference Optimization, DPO)

직접 선호 최적화(DPO)는 대규모 언어 모델(LLM)을 인간 선호도에 맞게 정렬하기 위해 특별히 설계된 더 최신의 방법입니다. 이 작업을 위해 PPO를 사용하는 것보다 더 단순하고 직접적인 대안을 제공합니다.

DPO를 이해하려면 먼저 전통적인 PPO 기반 정렬 방법을 이해하는 것이 도움이 됩니다.

- **PPO 접근법(2단계 프로세스)**:
  1. **보상 모델 훈련**: 먼저, 사람들이 다양한 LLM 응답을 평가하거나 비교하는 인간 피드백 데이터를 수집합니다(예: "응답 A가 응답 B보다 낫다"). 이 데이터는 별도의 AI 모델인 보상 모델(reward model)을 훈련하는 데 사용되며, 보상 모델의 역할은 새로운 응답에 대해 인간이 부여할 점수를 예측하는 것입니다.
  2. **PPO로 파인튜닝**: 그 다음, LLM은 PPO를 사용하여 파인튜닝됩니다. LLM의 목표는 보상 모델로부터 가능한 가장 높은 점수를 받는 응답을 생성하는 것입니다. 보상 모델은 훈련 게임에서 "심판" 역할을 합니다.

  이 2단계 프로세스는 복잡하고 불안정할 수 있습니다. 예를 들어, LLM이 허점을 찾아 나쁜 응답에 높은 점수를 받기 위해 보상 모델을 "해킹"하는 법을 배울 수도 있습니다.

- **DPO 접근법(직접 프로세스)**: DPO는 보상 모델을 완전히 건너뜁니다. 인간 선호도를 보상 점수로 변환한 다음 그 점수에 대해 최적화하는 대신, DPO는 선호 데이터를 직접 사용하여 LLM의 정책을 업데이트합니다.
- DPO는 선호 데이터를 최적 정책에 직접 연결하는 수학적 관계를 사용합니다. 본질적으로 모델에게 다음과 같이 가르칩니다: "선호되는 응답과 유사한 응답을 생성할 확률을 높이고 선호되지 않는 응답과 유사한 응답을 생성할 확률을 낮추어라."

결론적으로, DPO는 인간 선호 데이터에 직접 언어 모델을 최적화함으로써 정렬 프로세스를 단순화합니다. 이를 통해 별도의 보상 모델을 훈련하고 사용하는 복잡성과 잠재적 불안정성을 피해 정렬 프로세스를 보다 효율적이고 견고하게 만듭니다.

## 실용적 응용 및 사용 사례

적응형 에이전트는 경험적 데이터에 의한 반복적 업데이트를 통해 가변적인 환경에서 향상된 성능을 보입니다.

- **개인화된 어시스턴트 에이전트**는 개별 사용자 행동의 장기적 분석을 통해 상호작용 프로토콜을 개선하여 고도로 최적화된 응답 생성을 보장합니다.
- **거래 봇 에이전트**는 고해상도 실시간 시장 데이터를 기반으로 모델 파라미터를 동적으로 조정하여 의사결정 알고리즘을 최적화함으로써 금융 수익을 극대화하고 위험 요소를 완화합니다.
- **애플리케이션 에이전트**는 관찰된 사용자 행동을 기반으로 동적 수정을 통해 사용자 인터페이스와 기능을 최적화하여 사용자 참여도와 시스템 직관성을 높입니다.
- **로봇 및 자율주행 차량 에이전트**는 센서 데이터와 과거 행동 분석을 통합하여 탐색 및 반응 능력을 향상시킴으로써 다양한 환경 조건에서 안전하고 효율적인 운행을 가능하게 합니다.
- **사기 탐지 에이전트**는 새로 식별된 사기 패턴으로 예측 모델을 개선하여 이상 탐지 능력을 향상시키고 시스템 보안을 강화하며 금융 손실을 최소화합니다.
- **추천 에이전트**는 사용자 선호 학습 알고리즘을 사용하여 콘텐츠 선택 정밀도를 높임으로써 고도로 개인화되고 맥락적으로 관련성 있는 추천을 제공합니다.
- **게임 AI 에이전트**는 전략적 알고리즘을 동적으로 적응시켜 플레이어 참여도를 높이고 게임 복잡성과 도전을 증가시킵니다.
- **지식 베이스 학습 에이전트**: 에이전트는 검색 증강 생성(RAG)을 활용하여 문제 설명과 검증된 해결책의 동적 지식 베이스를 유지할 수 있습니다(14장 참조). 성공적인 전략과 발생한 어려움을 저장함으로써 에이전트는 의사결정 중 이 데이터를 참조하여 이전에 성공적인 패턴을 적용하거나 알려진 함정을 피함으로써 새로운 상황에 더 효과적으로 적응할 수 있습니다.

## 사례 연구: 자기 개선 코딩 에이전트(SICA)

Maxime Robeyns, Laurence Aitchison, Martin Szummer가 개발한 자기 개선 코딩 에이전트(Self-Improving Coding Agent, SICA)는 에이전트 기반 학습의 발전을 대표하며, 에이전트가 자신의 소스 코드를 수정할 수 있는 능력을 보여줍니다. 이는 한 에이전트가 다른 에이전트를 훈련시키는 전통적인 접근 방식과 대비됩니다. SICA는 수정자와 수정 대상 모두로서 작동하며, 다양한 코딩 과제에서 성능을 향상시키기 위해 반복적으로 코드 베이스를 개선합니다.

SICA의 자기 개선은 반복적인 사이클을 통해 이루어집니다(그림 1 참조). 처음에 SICA는 과거 버전들의 아카이브와 벤치마크 테스트에서의 성능을 검토합니다. 성공, 시간, 계산 비용을 고려한 가중 공식으로 계산된 가장 높은 성능 점수를 가진 버전을 선택합니다. 그런 다음 선택된 버전이 다음 자기 수정 라운드를 수행합니다. 아카이브를 분석하여 잠재적인 개선 사항을 파악하고 코드 베이스를 직접 수정합니다. 수정된 에이전트는 이후 벤치마크에 대해 테스트되며 결과는 아카이브에 기록됩니다. 이 과정이 반복되어 과거 성능으로부터 직접 학습이 이루어집니다. 이 자기 개선 메커니즘을 통해 SICA는 전통적인 훈련 패러다임 없이도 능력을 발전시킬 수 있습니다.

SICA는 상당한 자기 개선을 거쳐 코드 편집 및 탐색 측면에서 발전을 이루었습니다. 처음에 SICA는 코드 변경을 위해 기본적인 파일 덮어쓰기 방식을 사용했습니다. 이후 더 지능적이고 맥락적인 편집이 가능한 "스마트 에디터(Smart Editor)"를 개발했습니다. 이는 대상 수정을 위한 diff와 패턴 기반 편집을 통합한 "Diff 강화 스마트 에디터(Diff-Enhanced Smart Editor)"와 처리 요구 사항을 줄이기 위한 "빠른 덮어쓰기 도구(Quick Overwrite Tool)"로 진화했습니다.

SICA는 또한 효율성을 위해 추상 구문 트리(Abstract Syntax Tree, AST) 파싱을 사용하는 "최소 Diff 출력 최적화(Minimal Diff Output Optimization)"와 "맥락 민감 Diff 최소화(Context-Sensitive Diff Minimization)"를 구현했으며, "SmartEditor 입력 정규화기(SmartEditor Input Normalizer)"도 추가했습니다. 탐색 측면에서 SICA는 코드의 구조적 맵(AST)을 사용하여 코드 베이스 내의 정의를 식별하는 "AST 심볼 찾기(AST Symbol Locator)"를 독자적으로 생성했습니다. 이후 빠른 검색과 AST 확인을 결합한 "하이브리드 심볼 찾기(Hybrid Symbol Locator)"가 개발되었고, 관련 코드 섹션에 집중하여 검색 속도를 향상시키는 "하이브리드 심볼 찾기의 최적화된 AST 파싱(Optimized AST Parsing in Hybrid Symbol Locator)"을 통해 추가로 최적화되었습니다(그림 2 참조).

<figure><figcaption>다이어그램 ch09-fig2 (준비 중)</figcaption></figure>

SICA의 아키텍처는 기본 파일 작업, 명령 실행, 산술 계산을 위한 기초 툴킷으로 구성됩니다. 결과 제출 메커니즘과 특화된 하위 에이전트(코딩, 문제 해결, 추론) 호출 기능을 포함합니다. 이러한 하위 에이전트들은 복잡한 작업을 분해하고 특히 장기적인 개선 사이클 동안 LLM의 컨텍스트 길이를 관리합니다.

비동기 감독자인 또 다른 LLM이 SICA의 동작을 모니터링하여 루프나 정체 같은 잠재적 문제를 식별합니다. SICA와 소통하며 필요 시 실행을 중단하도록 개입할 수 있습니다. 감독자는 패턴과 비효율성을 파악하기 위해 콜 그래프와 메시지 및 도구 작업 로그를 포함한 SICA의 상세 행동 보고서를 받습니다.

SICA의 LLM은 작동에 있어 핵심적인 구조화된 방식으로 단기 기억인 컨텍스트 윈도우 내에서 정보를 구성합니다. 이 구조에는 에이전트 목표, 도구 및 하위 에이전트 문서, 시스템 지침을 정의하는 시스템 프롬프트가 포함됩니다. 핵심 프롬프트에는 문제 설명 또는 지시사항, 열린 파일의 내용, 디렉토리 맵이 담겨 있습니다. 어시스턴트 메시지에는 에이전트의 단계별 추론, 도구 및 하위 에이전트 호출 기록과 결과, 감독자 통신이 기록됩니다. 이러한 구성은 효율적인 정보 흐름을 촉진하여 LLM 작동을 향상시키고 처리 시간과 비용을 줄입니다. 초기에 파일 변경 사항은 수정 사항만 표시하는 diff로 기록되었으며 주기적으로 통합되었습니다.

### SICA: 코드 살펴보기

SICA의 구현을 더 깊이 살펴보면 그 기능을 뒷받침하는 몇 가지 핵심 설계 선택이 드러납니다. 앞서 논의한 바와 같이, 시스템은 모듈식 아키텍처로 구축되어 코딩 에이전트, 문제 해결 에이전트, 추론 에이전트 등 여러 하위 에이전트를 통합합니다. 이러한 하위 에이전트들은 도구 호출과 유사하게 주 에이전트에 의해 호출되어 복잡한 작업을 분해하고 특히 장기적인 메타 개선 반복 중 컨텍스트 길이를 효율적으로 관리합니다.

이 프로젝트는 활발하게 개발 중이며, 도구 사용 및 기타 에이전틱 작업에 대한 사후 훈련 LLM에 관심 있는 사람들을 위한 견고한 프레임워크를 제공하는 것을 목표로 합니다. 전체 코드는 GitHub 저장소(https://github.com/MaximeRobeyns/self_improving_coding_agent/)에서 추가 탐구 및 기여를 위해 공개되어 있습니다. 보안을 위해 이 프로젝트는 Docker 컨테이너화를 강하게 강조합니다. 에이전트가 셸 명령을 실행할 수 있다는 점에서 호스트 머신으로부터의 격리를 제공하여 의도치 않은 파일 시스템 조작 같은 위험을 완화합니다.

투명성과 제어를 보장하기 위해, 시스템은 이벤트 버스의 이벤트와 에이전트의 콜 그래프를 시각화하는 인터랙티브 웹 페이지를 통해 강력한 관측 가능성을 제공합니다. 이를 통해 에이전트의 행동에 대한 포괄적인 인사이트를 얻을 수 있으며, 사용자는 개별 이벤트를 검사하고, 감독자 메시지를 읽고, 더 명확한 이해를 위해 하위 에이전트 추적을 접을 수 있습니다.

핵심 지능 측면에서, 에이전트 프레임워크는 다양한 제공업체의 LLM 통합을 지원하여 특정 작업에 가장 적합한 모델을 찾기 위한 실험을 가능하게 합니다. 마지막으로 중요한 구성 요소는 비동기 감독자입니다. 이 LLM은 주 에이전트와 동시에 실행되며 에이전트의 동작에서 병리적 편차나 정체를 주기적으로 평가하고 필요 시 알림 전송이나 에이전트 실행 취소로 개입할 수 있습니다. 콜 그래프와 LLM 메시지, 도구 호출, 응답의 이벤트 스트림을 포함하는 시스템 상태의 상세한 텍스트 표현을 받아 비효율적인 패턴이나 반복 작업을 감지할 수 있습니다.

초기 SICA 구현에서 주목할 만한 과제 중 하나는 각 메타 개선 반복 동안 LLM 기반 에이전트가 독자적으로 새롭고 혁신적이며 실행 가능하고 흥미로운 수정을 제안하도록 유도하는 것이었습니다. 열린 학습과 LLM 에이전트의 진정한 창의성을 키우는 이 한계는 현재 연구에서 주요 탐구 영역으로 남아 있습니다.

## AlphaEvolve와 OpenEvolve

AlphaEvolve는 Google이 개발한 알고리즘을 발견하고 최적화하도록 설계된 AI 에이전트입니다. LLM, 특히 Gemini 모델(Flash 및 Pro), 자동화된 평가 시스템, 진화 알고리즘 프레임워크의 조합을 활용합니다. 이 시스템은 이론 수학과 실용 컴퓨팅 응용 분야 모두를 발전시키는 것을 목표로 합니다.

AlphaEvolve는 Gemini 모델 앙상블을 사용합니다. Flash는 광범위한 초기 알고리즘 제안 생성에 사용되고, Pro는 더 심층적인 분석과 개선을 제공합니다. 제안된 알고리즘은 사전 정의된 기준에 따라 자동으로 평가되고 점수가 매겨집니다. 이 평가는 솔루션을 반복적으로 개선하는 데 사용되는 피드백을 제공하여 최적화되고 새로운 알고리즘으로 이어집니다.

실용 컴퓨팅 분야에서 AlphaEvolve는 Google의 인프라에 배포되었습니다. 데이터 센터 스케줄링 개선으로 전 세계 컴퓨팅 자원 사용량을 0.7% 줄였습니다. 또한 차세대 텐서 처리 장치(Tensor Processing Units, TPU)의 Verilog 코드 최적화를 제안하여 하드웨기 설계에 기여했습니다. 나아가 AlphaEvolve는 Gemini 아키텍처의 핵심 커널에서 23% 속도 향상, FlashAttention의 저수준 GPU 명령어에서 최대 32.5% 최적화를 포함한 AI 성능을 가속화했습니다.

기초 연구 분야에서 AlphaEvolve는 48번의 스칼라 곱셈을 사용하는 4x4 복소수 행렬 방법을 포함하여 기존의 알려진 솔루션을 능가하는 새로운 행렬 곱셈 알고리즘 발견에 기여했습니다. 더 넓은 수학 연구에서는 75%의 경우 50개 이상의 미해결 문제에서 기존 최첨단 솔루션을 재발견하고 20%의 경우 기존 솔루션을 개선했으며, 키싱 수 문제(kissing number problem)에서의 발전 사례가 있습니다.

OpenEvolve는 LLM을 활용하여 코드를 반복적으로 최적화하는 진화적 코딩 에이전트입니다(그림 3 참조). LLM 기반 코드 생성, 평가, 선택의 파이프라인을 조율하여 다양한 작업에 대해 프로그램을 지속적으로 향상시킵니다. OpenEvolve의 핵심 특징은 단일 함수에 한정되지 않고 전체 코드 파일을 진화시키는 능력입니다. 에이전트는 다양성을 위해 설계되어 여러 프로그래밍 언어를 지원하고 모든 LLM에 대한 OpenAI 호환 API와 호환됩니다. 또한 다중 목표 최적화를 통합하고 유연한 프롬프트 엔지니어링을 허용하며 복잡한 코딩 과제를 효율적으로 처리하기 위한 분산 평가를 수행할 수 있습니다.

<figure><figcaption>다이어그램 ch09-fig3 (준비 중)</figcaption></figure>

다음 코드 스니펫은 OpenEvolve 라이브러리를 사용하여 프로그램에 대한 진화적 최적화를 수행합니다. 초기 프로그램 경로, 평가 파일, 설정 파일의 경로로 OpenEvolve 시스템을 초기화합니다. `evolve.run(iterations=1000)` 라인은 프로그램의 개선된 버전을 찾기 위해 1000회 반복으로 진화 프로세스를 시작합니다. 마지막으로 진화 중 발견된 최적 프로그램의 메트릭을 소수점 넷째 자리까지 형식화하여 출력합니다.

```python
from openevolve import OpenEvolve
```

```python
# Initialize the system
evolve = OpenEvolve(
   initial_program_path="path/to/initial_program.py",
   evaluation_file="path/to/evaluator.py",
   config_path="path/to/config.yaml"
)

# Run the evolution
best_program = await evolve.run(iterations=1000)
print(f"Best program metrics:")
for name, value in best_program.metrics.items():
   print(f"  {name}: {value:.4f}")
```

## 한눈에 보기

**무엇이 문제인가**: AI 에이전트는 종종 사전에 프로그래밍된 로직이 충분하지 않은 동적이고 예측 불가능한 환경에서 운영됩니다. 초기 설계 시 예상하지 못한 새로운 상황에 직면할 때 성능이 저하될 수 있습니다. 경험으로부터 학습하는 능력 없이는 에이전트가 전략을 최적화하거나 시간이 지남에 따라 상호작용을 개인화할 수 없습니다. 이러한 경직성은 효과를 제한하고 복잡한 실제 시나리오에서 진정한 자율성 달성을 방해합니다.

**왜 필요한가**: 표준화된 해결책은 학습 및 적응 메커니즘을 통합하여 정적 에이전트를 동적이고 진화하는 시스템으로 변환하는 것입니다. 이를 통해 에이전트는 새로운 데이터와 상호작용을 기반으로 자율적으로 지식과 행동을 개선할 수 있습니다. 에이전틱 시스템은 강화학습부터 자기 개선 코딩 에이전트(SICA)에서 볼 수 있는 자기 수정과 같은 더 고급 기법까지 다양한 방법을 사용할 수 있습니다. Google의 AlphaEvolve와 같은 고급 시스템은 LLM과 진화 알고리즘을 활용하여 복잡한 문제에 대한 완전히 새롭고 더 효율적인 솔루션을 자율적으로 발견합니다. 지속적인 학습을 통해 에이전트는 새로운 작업을 마스터하고 성능을 향상시키며 지속적인 수동 재프로그래밍 없이 변화하는 조건에 적응할 수 있습니다.

**경험 법칙**: 동적이고 불확실하거나 진화하는 환경에서 운영해야 하는 에이전트를 구축할 때 이 패턴을 사용하십시오. 개인화, 지속적인 성능 향상, 새로운 상황을 자율적으로 처리하는 능력이 필요한 응용 프로그램에 필수적입니다.

## 핵심 요점

- **학습과 적응**은 에이전트가 경험을 활용하여 하는 일을 더 잘하고 새로운 상황을 처리하는 것에 관한 것입니다.
- "적응"은 학습에서 비롯되는 에이전트의 행동이나 지식의 가시적인 변화입니다.
- SICA, 자기 개선 코딩 에이전트는 과거 성능을 기반으로 코드를 수정하여 자기 개선을 이룹니다. 이를 통해 스마트 에디터와 AST 심볼 찾기 같은 도구가 탄생했습니다.
- 특화된 "하위 에이전트"와 "감독자"를 두는 것은 이러한 자기 개선 시스템이 큰 작업을 관리하고 올바른 방향을 유지하는 데 도움을 줍니다.
- LLM의 "컨텍스트 윈도우"가 설정되는 방식(시스템 프롬프트, 핵심 프롬프트, 어시스턴트 메시지)은 에이전트가 얼마나 효율적으로 작동하는지에 매우 중요합니다.
- 이 패턴은 항상 변화하거나 불확실하거나 개인적인 접촉이 필요한 환경에서 운영되어야 하는 에이전트에게 필수적입니다.
- 학습하는 에이전트를 구축하는 것은 종종 머신러닝 도구와의 연결 및 데이터 흐름 관리를 의미합니다.
- 기본적인 코딩 도구를 갖춘 에이전트 시스템은 자율적으로 자신을 편집하여 벤치마크 작업에서 성능을 향상시킬 수 있습니다.
- AlphaEvolve는 LLM과 진화 프레임워크를 활용하여 알고리즘을 자율적으로 발견하고 최적화하는 Google의 AI 에이전트로, 기초 연구와 실용 컴퓨팅 응용 분야 모두를 크게 향상시킵니다.
